## 1. プロジェクト概要
指定したディレクトリ、およびその配下にある全てのサブディレクトリから、特定の条件に合致するExcelファイルを特定し、指定されたセルの値を抽出して整形出力する。

## 2. 技術スタック
- OS: Windows / macOS 両対応（`pathlib`を使用すること）
- 言語: Python 3.10以上
- 依存ライブラリ: `pandas`, `openpyxl` (Excel操作用), `configparser` (ini用)

## 3. コーディング規約
- エラーメッセージは日本語で記述
- エラーハンドリングは必ず行う
- 保守がしやすいように複雑な箇所はコメントを日本語で追加する
- enumは整数型で定義

## 4. 命名規則
- サービスクラス: 動詞 + 名詞 + Service（例: ExtractExcelCellService）
- クエリオブジェクト: 名詞 + Query（例: ExcelFileQuery）

## 5. 開発プロセス（テスト駆動開発）
- **Red-Green-Refactorサイクルを厳守すること**
  1. **Red**: 機能実装前に必ずテストコードを先に書く（この時点でテストは失敗する）
  2. **Green**: テストが通る最小限の実装を行う
  3. **Refactor**: テストが通ることを確認しながらリファクタリングを行う
- 新機能・修正を実装する際は、必ず以下の順序で進めること
  1. テストコードを作成
  2. テストを実行して失敗を確認
  3. 実装を行う
  4. テストを実行して成功を確認
  5. 必要に応じてリファクタリング
- テストフレームワーク: `pytest`を使用
- テストファイルの配置: `tests/` ディレクトリ配下に配置
- テストファイルの命名: `test_*.py` 形式
